# ROS 2 Core Concepts

This section covers the fundamental components of ROS 2: Nodes, Topics, Services, and Actions. These are the building blocks of all ROS 2 systems and form the foundation for more advanced concepts.

## Learning Objectives

By the end of this section, you will:

- Understand the core components of ROS 2 (Nodes, Topics, Services, Actions)
- Create and execute simple Publisher and Subscriber pairs in Python
- Verify data flow using topic echoing commands
- Appreciate the role of ROS 2 in enabling communication between different components

## Core Components Overview

### Nodes
A node is a process that performs computation. Nodes are the fundamental building blocks of a ROS 2 system. Each node can perform a specific task and communicate with other nodes through topics, services, or actions.

### Topics
Topics enable asynchronous communication between nodes through a publish/subscribe model. Publishers send messages to a topic, and subscribers receive messages from a topic.

### Services
Services enable synchronous request/response communication between nodes. A client sends a request to a service, and the service responds with a result.

### Actions
Actions enable asynchronous communication with feedback for long-running tasks. They provide goal, result, and feedback mechanisms for complex operations.
